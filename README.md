# BE-데일리 개인 회고 및 주간 프롬프트 회고 포맷 저장소 

DailyJob 활동 내용을 날짜별로 정리하는 회고 저장소입니다.

이 저장소의 회고는 단순히 작업량을 보고하기 위한 문서가 아니라, 활동 과정에서 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 기록하기 위한 용도입니다.

## 회고 종류

- 일일 회고: `dailyReview-template.md` 파일을 복제해서 작성합니다.
- 주간 회고: `weeklyPromptReview-template.md` 파일을 복제해서 작성합니다.

## 기록 방식

회고 파일은 날짜 기준 디렉터리 안에 Markdown 파일로 정리합니다.

```text
YYYY-MM-DD/
└── YYYY-MM-DD.md
```

예시는 다음과 같습니다.

```text
2026-06-29/
└── 2026-06-30.md
```

**주간 프롬프트 회고 내용은 금요일 Daily 회고 파일에 추가해서 작성합니다.**

새로운 회고를 작성할 때는 필요한 템플릿 파일을 복제한 뒤, 해당 날짜 디렉터리 안에 `.md` 파일로 저장합니다.

## 작성 흐름

1. 기록할 날짜에 맞는 `YYYY-MM-DD` 디렉터리를 만듭니다.
2. 일일 회고 또는 주간 회고 템플릿을 선택합니다.
3. 선택한 `*-template.md` 파일을 복제합니다.
4. 복제한 파일을 날짜 디렉터리 안에 `.md` 파일로 저장합니다.
5. 템플릿의 각 항목에 맞춰 활동 내용, 판단 근거, 문제 해결 과정, 다음 행동을 구체적으로 작성합니다.

## 작성 기준

- 각 항목은 길게 쓰기보다 사실, 판단, 근거, 다음 행동이 드러나게 작성합니다.
- 완료한 일뿐 아니라 막혔던 문제와 해결 과정을 함께 남깁니다.
- 잘한 행동과 아쉬웠던 행동은 다음 회고에서 반복하거나 개선할 수 있을 만큼 구체적으로 적습니다.
