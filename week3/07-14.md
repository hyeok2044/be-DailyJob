###### 3. 가장 막혔던 문제와 해결 과정

- 문제 상황 또는 증상: 코드를 JOIN을 하는 과정에서 ALIAS가 생겼다. 그 과정에서

    ```java
    String userId = rs.getString("c.USER_ID"); // Wrong
    String userId = rs.getString("USER_ID");   // Right
    ```

    아래와 같이 Column을 어떻게 가져와야 할지 생각해야 했다.

    이 과정에서 SQL exception을 catch 해주었기에 당연히 바깥으로 exception을 받아서 debug log가 나올 것이라 생각하고 다른 부분부터 디버깅을 시작하였다.

- 해결 방법: SQL Exception을 바깥에서 받아주는 부분이 없어서 로깅이 되지 않고 있었다. Log를 넣었다.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 꼭 문제가 생길 것 같은 부분에는 `logger.error`으로 정확한 증상을 알 수 있게 짜야겠다.

###### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)

- 비효율적이었던 접근/놓치거나 부족했던 점:

    Comment 객체를 불러 올 때 User의 정보를 불러오기 위해서는 DB access가 필요함.

    Map<String, User>를 받는 쿼리를 짜면 편할 것 같은데 User에 Password 객체도 있음.

    - UserWithoutCredentials 객체를 만드는게 낫나?
    - 아니면 데이터는 항상 서버에 있고 client에 전달은 안하니 User 객체 그대로 넘기는 것이 낫나?

    라는 고민을 했었다. 하지만 결국엔 객체를 다 만들고 JOIN을 통한 간단한 SQL 해법을 놓치게 되었다.

- 그렇게 된 원인: Java에 익숙하지 않아서 하나의 쿼리 → 하나의 객체를 사용하는 것에 매몰되어 PHP에서 항상 이용하던 JOIN 후 한 번에 값들을 가져오는 방식을 생각도 하지 못했다.
- 다음에는 다르게 해볼 구체적인 행동: 객체 지향적인 사고도 중요하지만 더욱 더 단순한 방법이 없는지 끊임없이 고민해보고, AI에 도움을 최대한 많이 청하고 기록하여 발전해야 한다.
