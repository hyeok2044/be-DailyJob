# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 웹서버 3단계 제출
- (오늘의 목표 중에서) 아직 끝내지 못한 부분: PR review가 pass되도록 코드 고치기

###### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: Request Target을 객체로 만든 것
- 그렇게 판단한 이유: 코드가 짧아지고, 유지보수가 쉬워진 것 같음.
- 고려했지만 선택하지 않은 대안: in-line parsing
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 코드의 품질과 테스팅이 용이해졌다. 다만 슬슬 클래스들이 많아지는 것 같은데 명명법을 조금 더 확실히 해야 내가 알아보기 쉬울 것 같다.

###### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상:
- 처음 추측한 원인:
- 원인을 확인하기 위해 시도한 방법:
- 실제 원인:
- 해결 방법:
- 다음에 비슷한 문제가 생기면 먼저 확인할 것:

> 어제 3단계를 많이 구현해서 막혔던 문제 없이 순조롭게 진행된 것 같다.

###### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: TDD 원칙을 열심히 지켜서 개발하였다.
- 그 접근이 도움이 된 이유: parsing과 같은 헷갈리는 구현에 대해서 안정감 있게 코딩
- 다음에도 반복하고 싶은 점: 좋은 코딩 습관들을 체득하고 싶다.

###### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: commit 단위를 잘게 쪼갤 생각을 하지 못했음.
- 그렇게 된 원인: notion 문서로 개발 문서 형식으로 적어놓아서 예전에 어떤 생각을 하면서 코드를 작성할 지 생각하지 못한 것 같다.
- 다음에는 다르게 해볼 구체적인 행동: commit 단위를 잘게 쪼개서 의미 있는 commit log를 통해서 버전 컨트롤을 용이하게 할 것이다.
